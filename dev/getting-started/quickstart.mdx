Get up and running with Osmedeus in 5 minutes.

## Prerequisites

- Go 1.21 or later
- Git

## Installation

```bash
# Clone the repository
git clone https://github.com/osmedeus/osmedeus.git
cd osmedeus

# Build the binary
make build

# The binary is created at bin/osmedeus
./bin/osmedeus --help
```

To install globally:

```bash
make install
# Now you can run: osmedeus --help
```

## Your First Workflow

Create a simple workflow file `hello.yaml`:

```yaml
kind: module
name: hello-world
description: A simple hello world workflow

params:
  - name: target
    required: true
    description: Target to greet

steps:
  - name: greet
    type: bash
    command: echo "Hello, {{target}}!"

  - name: timestamp
    type: bash
    command: echo "Current time:" && date
```

Run it:

```bash
osmedeus run -m hello.yaml -t "World"
```

## Running Built-in Workflows

List available workflows:

```bash
osmedeus workflow list
```

Show workflow details:

```bash
osmedeus workflow show <workflow-name>
```

Run a workflow:

```bash
# Run a module workflow
osmedeus run -m subdomain-enum -t example.com

# Run a flow workflow (orchestrates multiple modules)
osmedeus run -f general -t example.com
```

## Common Options

```bash
# Dry-run mode (preview without executing)
osmedeus run -m subdomain-enum -t example.com --dry-run

# Verbose output
osmedeus run -m subdomain-enum -t example.com -v

# Custom parameters
osmedeus run -m nuclei-scan -t example.com -p 'templates=cves'

# Multiple targets
osmedeus run -m subdomain-enum -t target1.com -t target2.com

# Targets from file
osmedeus run -m subdomain-enum -T targets.txt

# Concurrent scanning
osmedeus run -m subdomain-enum -T targets.txt -c 5
```

## Workspace Output

Scan results are stored in workspaces:

```
~/osmedeus-base/workspaces/
└── example.com/
    ├── subdomain-enum/
    │   └── subdomains.txt
    ├── http-probe/
    │   └── live-hosts.txt
    └── ...
```

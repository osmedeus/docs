Execute workflows against targets.

## Synopsis

```bash
osmedeus run [flags]
```

Aliases: `scan`

## Flags

### Required (one of)

| Flag | Description |
|------|-------------|
| `-f, --flow <name>` | Flow workflow to execute |
| `-m, --module <name>` | Module workflow(s) to execute (repeatable) |

### Targets

| Flag | Description |
|------|-------------|
| `-t, --target <target>` | Target(s) to scan (repeatable) |
| `-T, --target-file <file>` | File containing targets (one per line) |

### Parameters

| Flag | Description |
|------|-------------|
| `-p, --params <key=value>` | Additional parameters (repeatable) |
| `-P, --params-file <file>` | Load parameters from JSON/YAML file |

### Execution Control

| Flag | Description |
|------|-------------|
| `--dry-run` | Preview without executing |
| `--threads-hold <int>` | Override thread count |
| `-B, --tactic <name>` | Run tactic: `aggressive`, `default`, `gently` |
| `-x, --exclude <module>` | Module(s) to exclude (repeatable) |
| `-c, --concurrency <int>` | Concurrent targets (default: 1) |
| `--timeout <duration>` | Run timeout (e.g., `2h`, `3h`, `1d`) |
| `--repeat` | Repeat run after completion |
| `--repeat-wait-time <duration>` | Wait between repeats (default: `1h`) |

### Workspace

| Flag | Description |
|------|-------------|
| `-w, --workspace <name>` | Custom workspace name |
| `-W, --workspaces-folder <path>` | Override workspaces directory |
| `--clean-up-workspace` | Delete workspace after completion |
| `--disable-workspace-creation` | Skip workspace creation |

### Distributed Mode

| Flag | Description |
|------|-------------|
| `-D, --distributed-run` | Submit to distributed queue |
| `--redis-url <url>` | Redis URL for distributed mode |

### Output

| Flag | Description |
|------|-------------|
| `-v, --verbose` | Verbose output |
| `-G, --progress-bar` | Show progress bar |
| `--log-file <path>` | Log to file |
| `--log-file-tmp` | Create timestamped log file |

## Examples

### Basic Usage

```bash
# Run a flow
osmedeus run -f general -t example.com

# Run a module
osmedeus run -m subdomain-enum -t example.com

# Run multiple modules in sequence
osmedeus run -m subdomain -m portscan -m vulnscan -t example.com
```

### Multiple Targets

```bash
# Multiple targets via flags
osmedeus run -m scan -t target1.com -t target2.com -t target3.com

# Targets from file
osmedeus run -f general -T targets.txt

# Combine methods
osmedeus run -m scan -t main.com -T additional.txt

# From stdin
cat domains.txt | osmedeus run -m http-probe
echo "extra.com" | osmedeus run -m scan -t main.com
```

### Concurrent Scanning

```bash
# Scan 5 targets concurrently
osmedeus run -m recon -T targets.txt -c 5
```

### Parameters

```bash
# Single parameter
osmedeus run -m nuclei -t example.com -p 'templates=cves'

# Multiple parameters
osmedeus run -m scan -t example.com -p 'threads=20' -p 'severity=critical'

# From file (params.yaml: {templates: "cves", severity: "critical"})
osmedeus run -m nuclei -t example.com -P params.yaml
```

### Execution Options

```bash
# Dry run - preview execution
osmedeus run -f general -t example.com --dry-run

# Override threads
osmedeus run -f general -t example.com --threads-hold 100

# Use gentle tactic (fewer threads)
osmedeus run -f general -t example.com -B gently

# Exclude modules
osmedeus run -f general -t example.com -x screenshot -x portscan
```

### Timeouts and Repeats

```bash
# Cancel if exceeds 2 hours
osmedeus run -m recon -t example.com --timeout 2h

# Repeat continuously every hour
osmedeus run -m recon -t example.com --repeat --repeat-wait-time 1h
```

### Distributed Mode

```bash
# Submit to distributed queue
osmedeus run -f general -t example.com -D

# Custom Redis URL
osmedeus run -f general -t example.com -D --redis-url redis://redis-host:6379
```

### Logging

```bash
# Verbose output
osmedeus run -m scan -t example.com -v

# Progress bar (suppresses logs)
osmedeus run -m scan -t example.com -G

# Log to file
osmedeus run -m scan -t example.com --log-file /tmp/scan.log

# Auto-named log file
osmedeus run -m scan -t example.com --log-file-tmp
```

### Custom Workspace

```bash
# Custom workspace name
osmedeus run -m scan -t example.com -w my-custom-scan

# Clean up after completion
osmedeus run -m scan -t example.com --clean-up-workspace
```

## Global Flags

Available for all commands:

```bash
-b, --base-folder <path>      Override base folder
-F, --workflow-folder <path>  Override workflow folder
--settings-file <path>        Custom settings file
--debug                       Debug mode
-q, --silent                  Silent mode (errors only)
--disable-color               No colored output
--disable-notification        No notifications
--disable-db                  Disable database
--ci-output-format            JSON output for CI
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | General error |
| 2 | Invalid arguments |
| 124 | Timeout exceeded |

## Notes

- Targets can come from `-t`, `-T`, stdin, or pipeline
- Multiple `-m` flags run modules sequentially
- Use `--dry-run` to preview before execution
- Progress bar (`-G`) suppresses detailed logs and LLM output
- Distributed mode requires Redis

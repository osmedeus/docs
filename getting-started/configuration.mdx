Osmedeus configuration is managed through `osm-settings.yaml`.

## Configuration File

Default location: `~/osmedeus-base/osm-settings.yaml`

Override with:
```bash
osmedeus run -m example -t target --settings-file /path/to/settings.yaml
```

## Full Configuration Reference

```yaml
# Base folder containing all osmedeus data
base_folder: "{{HOME}}/osmedeus-base"

# Environment paths (relative to base_folder)
environments:
  binaries: "{{base_folder}}/binaries"
  data: "{{base_folder}}/data"
  workflows: "{{base_folder}}/workflows"
  workspaces: "{{base_folder}}/workspaces"
  storages: "{{base_folder}}/storages"

# Database configuration
database:
  db_engine: sqlite          # sqlite or postgres
  db_path: "{{base_folder}}/database-osm.sqlite"

  # PostgreSQL options (when db_engine: postgres)
  host: localhost
  port: 5432
  username: osmedeus
  password: secret
  db_name: osmedeus

# Server configuration
server:
  host: "0.0.0.0"
  port: 8002
  ui_path: "{{base_folder}}/ui/"
  workspace_prefix: ""       # Auto-generated 16-char random string
  username: osmedeus
  password: [REDACTED:password]

# Scan tactics (thread counts)
scan_tactic:
  aggressive:
    threads: 100
    base_threads: 50
  default:
    threads: 50
    base_threads: 25
  gently:
    threads: 10
    base_threads: 5

# Redis configuration (for distributed mode)
redis:
  url: "redis://localhost:6379"
  password: ""
  db: 0

# Notification settings
notification:
  telegram:
    enabled: false
    bot_token: ""
    chat_id: ""

# Cloud storage (S3-compatible)
storage:
  enabled: false
  provider: s3              # s3 or minio
  bucket: osmedeus-storage
  region: us-east-1
  endpoint: ""              # Custom endpoint for MinIO
  access_key: ""
  secret_key: ""

# LLM configuration
llm:
  provider: openai          # openai, anthropic, ollama
  api_key: ""
  model: gpt-4
  base_url: ""              # Custom endpoint
  temperature: 0.7
  max_tokens: 4096
```

## Environment Variables

Override any setting with environment variables:

```bash
# Server settings
export OSM_SERVER_HOST=0.0.0.0
export OSM_SERVER_PORT=8080
export OSM_SERVER_USERNAME=admin
export OSM_SERVER_PASSWORD=secret

# Database
export OSM_DATABASE_DB_ENGINE=postgres
export OSM_DATABASE_HOST=db.example.com
export OSM_DATABASE_PASSWORD=secret

# Redis
export OSM_REDIS_URL=redis://redis.example.com:6379

# LLM
export OSM_LLM_API_KEY=sk-...
```

## Common Configurations

### Change Default Credentials

```yaml
server:
  username: your-username
  password: your-secure-password
```

### Use PostgreSQL

```yaml
database:
  db_engine: postgres
  host: localhost
  port: 5432
  username: osmedeus
  password: your-password
  db_name: osmedeus
```

### Enable Telegram Notifications

```yaml
notification:
  telegram:
    enabled: true
    bot_token: "123456:ABC-DEF..."
    chat_id: "-1001234567890"
```

### Configure S3 Storage

```yaml
storage:
  enabled: true
  provider: s3
  bucket: my-osmedeus-bucket
  region: us-west-2
  access_key: [REDACTED:aws-access-key-id]
  secret_key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

### Use MinIO

```yaml
storage:
  enabled: true
  provider: minio
  bucket: osmedeus
  endpoint: http://minio.local:9000
  access_key: minioadmin
  secret_key: minioadmin
```

### Configure LLM

```yaml
llm:
  provider: openai
  api_key: sk-your-api-key
  model: gpt-4
```

For Ollama (local):

```yaml
llm:
  provider: ollama
  base_url: http://localhost:11434
  model: llama2
```

## Scan Tactics

Control thread counts with tactics:

```bash
# Use aggressive tactic (more threads)
osmedeus run -m recon -t target -B aggressive

# Use gentle tactic (fewer threads)
osmedeus run -m recon -t target -B gently

# Override threads directly
osmedeus run -m recon -t target --threads-hold 100
```

## Workspace Prefix

The `workspace_prefix` setting enables direct HTTP access to workspace files:

```
http://host:port/ws/{workspace_prefix}/target-folder/...
```

If empty, a random 16-character string is auto-generated on first run.

## CLI Global Flags

Override configuration via CLI:

```bash
# Custom base folder
osmedeus run -m example -t target -b /custom/base

# Custom workflow folder
osmedeus run -m example -t target -F /custom/workflows

# Custom settings file
osmedeus run -m example -t target --settings-file /path/to/settings.yaml

# Disable database
osmedeus run -m example -t target --disable-db

# Disable notifications
osmedeus run -m example -t target --disable-notification
```

## View Current Configuration

Via API:

```bash
curl http://localhost:8002/osm/api/settings/yaml \
  -H "Authorization: Bearer <token>"
```
